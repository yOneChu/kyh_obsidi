## 🏗️ JAVA-PID 작동 원리 (로직 에디터 수행 방식)

팀원들에게 설명할 핵심 내용을 다음 두 가지 주요 단계로 정리할 수 있습니다.

### 1. ⚙️ 로직 에디터의 기본 데이터 처리 방식: KEY-VAL 저장소

로직 에디터(Logic Editor)에서 수행되는 모든 계산 로직(BOM 계산, PID 시뮬레이션, 블록 시뮬레이션 등)은 다음과 같은 방식으로 데이터를 관리하고 사용합니다.

- **저장 방식:** 모든 **영업 사양값**들을 **Key-Value 쌍**의 형태로 저장합니다.
    
    - 이는 마치 **Python의 딕셔너리(Dictionary)** 또는 **JAVA의 Map** 자료구조와 유사합니다.
        
- **데이터 접근:** 로직이 특정 계산을 수행할 때, 필요한 **값(Value)**을 해당 **키(Key)**가 일치하는 저장소에서 찾아 꺼내어 계산에 사용합니다.
    
    - **예시:** `{"층수": 10, "도어_타입": "2S", "속도": 1.75}`
        

---

### 2. 🚀 JAVA-PID 로직 수행 흐름: Key-Val 기반 함수 호출

JAVA-PID 로직 역시 위에서 설명한 로직 에디터의 기본 데이터 처리 방식을 따르며, **Key-Value 쌍**을 활용하여 JAVA 함수를 호출하고 결과를 반환받습니다.

|**단계**|**설명**|
|---|---|
|**준비 (입력)**|1. JAVA 함수를 호출하기 **전**에, 해당 함수에서 사용할 **모든 변수**들을 Key-Value 쌍으로 **셋팅**하여 준비합니다. (즉, 함수의 입력 데이터 환경을 **Key-Val 저장소** 형태로 구성)|
|**실행 (처리)**|2. 셋팅된 Key-Value 환경을 가지고 **CALL-JAVA 함수**를 수행(호출)합니다.|
|**결과 (출력)**|3. JAVA 함수 수행이 완료되면, 미리 **약속된 형식**의 **리턴값**을 **Key-Value 쌍**의 형태로 받습니다.|

> **핵심 요약:** JAVA-PID는 함수 내에서 사용할 데이터를 미리 Key-Value(Map) 형태로 준비해두고, 이 데이터를 기반으로 JAVA 함수를 호출하며, 결과 역시 Key-Value 형태로 반환받아 로직 에디터 환경과 연동하는 방식입니다.
