### 2. 최초설계완료BOM

2.1 장점
> 1) 사양이 변경되었을 때 이전 사양으로 수배된 자재가 뭐인지 확인할 때.
> 2) JQPR 발생하였을 때, 해당 자재가 수정된 자재인 경우 최초설계완료BOM 대비 내가 뭐를 수정했는지 확인 용도.
> 3) 재고소진으로 수배로직에서 자재를 변경하였을 때, 원래 자재번호가 뭐였는지 확인할 때

**최초설계완료BOM**은 설계 완료 시점의 BOM을 기준 정보로 보관하여,  
이후 발생하는 자재 변경 이력을 비교·추적할 수 있도록 하는 기준 BOM입니다.

#### 1. 사양 변경 시 기존 수배 자재 확인 가능

제품 사양이 변경되면, 변경 이전 사양을 기준으로 이미 수배된 자재가 무엇인지 확인해야 할 필요가 있습니다.  
최초설계완료BOM이 있으면 **초기 설계 기준 자재와 현재 자재를 비교**할 수 있어,  
**기존에 어떤 자재가 수배되었는지 명확하게 추적**할 수 있습니다.

#### 2. JQPR 발생 시 변경 자재 이력 파악 가능

JQPR 발생 시 대상 자재가 변경된 자재라면,  
최초설계완료BOM을 기준으로 **초기 자재 대비 어떤 부분을 수정했는지 확인**할 수 있습니다.  
이를 통해 **변경 사유 분석, 품질 이슈 원인 파악, 변경 영향도 검토**가 쉬워집니다.

#### 3. 대체 자재 적용 시 원자재 정보 확인 가능

재고 소진 등의 사유로 수배 로직에서 자재를 다른 자재로 변경한 경우,  
최초설계완료BOM이 있으면 **원래 적용되어야 했던 자재번호를 확인**할 수 있습니다.  
즉, 대체 자재 운영 과정에서도 **원 기준 자재를 추적할 수 있어 관리 신뢰성이 높아집니다.**

2) 개발 사유

- 각 품목별 최초 BOM 수배이력을 관리할 수 있도록 기능 개발이 필요- 해당 이력은 추후 AS-IS / TO-BE 비교 시 기준 데이터로 활용하고, BOM 변경 여부 및 최초 수배 시점을 확인하기 위한 목적으로 사용하고자 합니다.

2) 개발 목적

•각 품목별 최초 BOM 수배이력을 저장 및 관리

•최초 수배 기준 데이터 확보하여 이력 추적 가능

•향후 AS-IS / TO-BE 비교 기능의 기초 데이터로 활용

•

3) 기능 요청 사항

•BOM 계산 시점에 각 품목별 최초 BOM 이력을 저장하는 기능 개발 요청

4) 상세 요구사항

4.1) 이력 저장 조건

•BOM 계산 수행 시, 각 품목별 최초 BOM 이력을 저장

•품목에 해당하는 자재가 1개라도 존재할 경우, 해당 품목에 대한 최초 이력을 저장

•이미 최초 이력이 저장된 품목은 추가로 저장하지 않음

•대상 품목: M, C, 1, 2, 3

4.2) 저장 종료 조건

대상 품목 M, C, 1, 2, 3에 대한 최초 BOM 이력이 모두 저장된 경우  
이후에는 추가 저장이 발생하지 않도록 처리

5) 삭제 조건

제품을 삭제할 경우, 해당 제품의 ‘BOM계산’ 수행한 이력은 지워야 함.

그러나 모든 품목의 이력이 저장된 이후의 제품 버전부터는 제품 삭제 시, 이력테이블 건들지 않는다.
