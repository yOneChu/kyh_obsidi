### 2. 최초설계완료BOM
BOM 계산할 때,



1) 개발 사유

- 각 품목별 최초 BOM 수배이력을 관리할 수 있도록 기능 개발이 필요- 해당 이력은 추후 AS-IS / TO-BE 비교 시 기준 데이터로 활용하고, BOM 변경 여부 및 최초 수배 시점을 확인하기 위한 목적으로 사용하고자 합니다.

2) 개발 목적

•각 품목별 최초 BOM 수배이력을 저장 및 관리

•최초 수배 기준 데이터 확보하여 이력 추적 가능

•향후 AS-IS / TO-BE 비교 기능의 기초 데이터로 활용

•

3) 기능 요청 사항

•BOM 계산 시점에 각 품목별 최초 BOM 이력을 저장하는 기능 개발 요청

4) 상세 요구사항

4.1) 이력 저장 조건

•BOM 계산 수행 시, 각 품목별 최초 BOM 이력을 저장

•품목에 해당하는 자재가 1개라도 존재할 경우, 해당 품목에 대한 최초 이력을 저장

•이미 최초 이력이 저장된 품목은 추가로 저장하지 않음

•대상 품목: M, C, 1, 2, 3

4.2) 저장 종료 조건

대상 품목 M, C, 1, 2, 3에 대한 최초 BOM 이력이 모두 저장된 경우  
이후에는 추가 저장이 발생하지 않도록 처리

5) 삭제 조건

제품을 삭제할 경우, 해당 제품의 ‘BOM계산’ 수행한 이력은 지워야 함.

그러나 모든 품목의 이력이 저장된 이후의 제품 버전부터는 제품 삭제 시, 이력테이블 건들지 않는다.
