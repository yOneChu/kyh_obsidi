## 01. 기능
이 화면(PLM 제품-자재 Analytics)은 PLM에 등록된 제품과 자재(BOM) 연계 데이터를 다양한 조건으로            
  조회·집계하는 대시보드입니다. 코드를 분석한 결과, 사용자가 얻을 수 있는 인사이트를 정리하면 다음과        
  같습니다. 
### 1. 특정 자재(Part)가 어떤 제품들에 쓰이는지 — 역추적 인사이트                                           
  도움말(searchPartAnalysis.html:731)과 searchPID() 로직상, 자재번호(PartNo)를 입력하면 그 자재가 사용된    
  모든 제품을 역으로 조회합니다.                                                                            
  - "이 부품을 단종/변경하면 영향받는 제품이 몇 종인가?" → 설계변경(ECO) 영향 범위 파악                     
  - BlockNo 기준으로도 조회 가능해, 특정 블록 단위가 어떤 제품군에 걸쳐 있는지 확인                         
  - 통계 패널의 고유 제품 수(statProducts, JS 394행)로 영향 제품 종수를 한눈에 집계                         
                                                                                                            
### 2. 제품 사양별 분포 인사이트 (브랜드 / 기종 / 속도)                                                       
                                                                                                            
  조회 후 통계 패널(renderStats(), JS 382행~)이 자동으로 집계해 줍니다.                                     
  - 브랜드 분포 / 기종(EL_ATYP) 분포: 이 자재·블록이 어느 브랜드·기종에 집중되어 있는지 (countDist())       
  - 속도(aspd) 구간 분포: 25 m/min 단위 버킷으로 막대 그래프화(439행) → 고속/저속 엘리베이터 중 어디에 주로 
  쓰이는지                                                                                                  
  - 평균 속도 / 속도 범위(min~max) KPI → 적용 제품의 속도 스펙 대역 파악                                    
                                                                                                            
### 3. 다축 조건 필터링을 통한 사양 교차 분석                        
  검색 조건이 생산거점(EL_ASPSCD), 관통(EL_ETHRU), 전망타입(EL_COB), WALL 구조(EL_BWALLT), 최초설계일(EL_ZFDA, 비교연산자 지원) 등으로 세분화되어 있고, 동적 조건(Key–연산자–Value) 까지 추가할 수 있습니다(addKeyValueRow()).
- 예: "국내+상해 생산거점(KC01) + 관통형 + 2026년 이후 설계" 제품에서 특정 자재 사용 현황
- Like/Not Like/Equal/Not Equal 연산으로 임의 속성 기준의 유연

### 4. 설계 시점(최초설계일) 기반 시계열·신구 비교                                                            
                                                                                                            
  EL_ZFDA에 >=, <, = 등 비교연산자를 적용할 수 있어:                                                        
  - "최근 설계된 제품 vs 과거 설계 제품"에서 동일 자재의 채택 추세 비교                                     
  - 신규 설계에 여전히 구형 부품이 쓰이는지(단종 대상 부품 잔존 여부) 확인                                  
                                                                                                            
### 5. 데이터 내보내기·재가공 인사이트                                                                        
                                                                                                    
  DataTable에 CSV/Excel/Copy 내보내기(initDataTable(), JS 330행)와 컬럼 토글(25개 컬럼)이 있어, 조회 결과를 
  외부 분석 도구(Excel 등)로 가져가 2차 피벗/원가·물량 분석이 가능합니다. QTY·용량(acapa)·CWT폭 등 정량     
  컬럼이 포함되어 자재 소요량 집계에도 활용됩니다.                                                          
                                                                                                            
  ---                                                                                                       
  정리하면, 이 화면은 ① 부품→제품 역추적(영향분석), ② 사양 분포 통계, ③ 다축 교차 필터링, ④ 설계시점 기반   
  추세 비교, ⑤ 데이터 추출·재가공 의 인사이트를 제공합니다.



## 02. 그래프
두 팝업(그래프 → searchPartGraph.html, 인터렉티브 → searchInteractive.html)을 분석했습니다. 둘 다 같은    
  조회 결과를 시각화하지만, "그래프"는 Highcharts 2D, "인터렉티브"는 3D 도넛·3D 컬럼 + Three.js 배경의      
  화려한 버전입니다. 집계 로직(renderAll())은 동일합니다. 여기서 얻을 수 있는 인사이트를 정리합니다.        
                                                                                                            
### 1. 한눈에 보는 규모 요약 (5대 KPI)                                                                        
                                                                                                            
  상단 KPI 카드(renderAll(), graph 490행 / interactive 576행)에서 즉시 파악:                                
  - 전체 건수 / 고유 제품 수(종) / 고유 자재 수(개) / 브랜드 수 / 평균 속도(m/min)                          
  - 예: "이 부품은 12개 브랜드, 340종 제품에 8,500건 쓰이고, 평균 105 m/min 제품에 적용" → 부품의 적용      
  폭(범용성) 을 단번에 판단                                                                                 
                                                                                                            
###  2. 사양별 점유율·쏠림 분석 (6종 분포 차트)                                                                
                                                                                                            
  브랜드 / 기종 / 속도구간 / 생산거점 / 전망타입 / 관통·WALL 구조 분포를 차트로(countBy() 집계):            
  - 브랜드 분포 도넛: 어느 브랜드에 집중되는지 점유율(%)                                                    
  - 기종 분포(상위 12/10): 주력 기종 vs 비주력 기종                                                         
  - 속도 구간 추세(25 단위 버킷): 저속/중속/고속 어디에 분포가 몰리는지                                     
  - 생산거점(EL_ASPSCD) 분포: 국내·상해·인도·US 등 거점별 적용 현황 → 공급망/현지화 인사이트                
  - 관통·전망타입·WALL 구조: 사양 옵션별 채택 비율           


### 3. 브랜드별 부품 구성(Mix) 비교 — 누적 막대

"브랜드별 부품 구성" 누적 막대(renderBrandPartChart())는 상위 브랜드 × 상위 부품(품명) 매트릭스를 쌓아 보여줍니다.
- "A 브랜드는 X부품 위주, B 브랜드는 Y부품 위주" 같은 브랜드 간 부품 구성 차이를 비교
- 상위 12개 외는 '기타'로 묶어 핵심 부품에 집중


### 4. 부품 표준화·공용화 기회 발굴 (집계 테이블)

- 품명별 사용 건수 Top 15(renderPartSummary()): 가장 많이 쓰이는 품목 → 표준품/공용화 후보
- 브랜드별 사용 부품 목록(renderBrandPartTable()): "사용 건수" + "사용 제품수(prodCnt)" 컬럼으로 → 여러 제품에 두루 쓰이는 핵심 부품 식별
- 브랜드 필터 칩 + 품번/품명/SPEC 실시간 검색으로 특정 브랜드·부품만 좁혀 분석

### 5. 자재 부하가 큰 제품 식별 (제품별 자재 건수 Top 15)

renderProductSummary() 는 자재 건수가 많은 제품 순위(브랜드·기종 동반 표시)를 보여줍니다.
- "어떤 제품이 가장 복잡한(자재 종류가 많은) 구성인가?" → 원가·설계 복잡도 관리 대상 파악
- 인터렉티브 버전은 상위 3위에 메달 강조(rank top)로 우선순위 시각화

정리 — 인사이트 한 줄 요약

1. 규모/범용성: KPI 5종으로 부품 적용 폭 즉시 파악
2. 사양 쏠림: 브랜드·기종·속도·거점·옵션 분포로 집중도 분석
3. 브랜드 Mix 비교: 누적 막대로 브랜드별 부품 구성 차이
4. 표준화 후보 발굴: 품명별·제품수 기준 공용 부품 식별
5. 복잡 제품 식별: 자재 건수 Top 제품으로 원가/복잡도 관리
