

웹 서버는 사진이나 텍스트, 동영상 파일처럼 요청에 따라 내용이 달라지지 않는 데이터를 처리할 수 있으며, 이러한 데이터를 가리켜 정적(static) 데이터 라고 합니다.

정적데이터는 클라이언트가 요청할 때마다(파일이 바뀌지 않았다면) 결과값이 동일한 데이터 입니다.

이러한 웹 서버의 종류로는 NGINX, Apache 등이 있습니다.

만약에 클라이언트가 현재 로그인한 사용자의 계정정ㅂ조를 요청하면 어떻게 될까요?
웹서버는 이 데이터를 처리할 수 없습니다. 왜냐하면 웹 서버는 일반적으로 로그인 정보 조회 등 사용자의 요청에 따라 바뀔 수 있는 데이터를  처리할 수 없기 때문입니다.

이러한 데이터를 동적(dynamic) 데이터라고 합니다.

# 웹 서버란 무엇인가: 개발 입문자를 위한 기초 개념 정리

개발 공부를 시작하거나 IT 직무로 커리어를 준비하다 보면, "웹 서버"라는 용어를 정말 자주 마주치게 됩니다. 강의에서도, 채용 공고에서도, 면접 질문에서도 빠지지 않는 단어죠. 하지만 막상 "웹 서버가 무엇인가요?"라는 질문을 받으면, 명확하게 설명하기 쉽지 않습니다.

이 글에서는 웹 서버의 개념과 동작 원리를 차근차근 정리해보려고 합니다. 처음 공부하시는 분들도, 다시 복습이 필요한 분들도 부담 없이 따라오실 수 있도록 구성했습니다.

## 1. 웹 서버의 정의

웹 서버(Web Server)는 두 가지 의미로 사용됩니다.

첫 번째는 **소프트웨어**로서의 웹 서버입니다. 클라이언트(주로 웹 브라우저)의 HTTP 요청을 받아, 정적 콘텐츠(HTML, CSS, JavaScript, 이미지 등)를 응답으로 전달하는 프로그램을 말합니다. 두 번째는 **하드웨어**로서의 웹 서버입니다. 이 소프트웨어가 설치되어 실제로 작동하는 컴퓨터 자체를 가리키기도 합니다.

문맥에 따라 두 의미가 혼용되지만, 개발 공부 초반에는 "요청을 받아 응답을 돌려주는 프로그램"으로 이해하시면 충분합니다.

## 2. 클라이언트-서버 모델 이해하기

웹 서버를 이해하려면 먼저 **클라이언트-서버 모델**을 알아야 합니다.

우리가 브라우저 주소창에 `www.example.com`을 입력하는 순간, 브라우저는 해당 도메인에 연결된 서버로 요청(Request)을 보냅니다. 서버는 그 요청을 처리한 뒤 응답(Response)을 돌려주고요. 브라우저는 받은 응답을 해석해서 화면에 페이지를 그려냅니다.

이때 요청을 보내는 쪽이 **클라이언트**, 요청을 받아 응답하는 쪽이 **서버**입니다. 웹 서버는 이 구조에서 "서버" 역할을 담당하는 핵심 구성 요소입니다.

## 3. HTTP라는 통신 규약

클라이언트와 웹 서버는 아무렇게나 대화하지 않습니다. **HTTP(HyperText Transfer Protocol)** 라는 정해진 규약에 따라 메시지를 주고받습니다. 보안이 강화된 버전은 HTTPS이고요.

HTTP 요청에는 메서드(GET, POST, PUT, DELETE 등)와 경로, 헤더 등이 포함됩니다. 서버는 이를 해석한 뒤 상태 코드와 함께 응답을 보냅니다. 우리가 흔히 보는 응답 코드는 다음과 같습니다.

- **200 OK**: 요청이 성공적으로 처리됨
- **301 Moved Permanently**: 요청한 리소스가 영구적으로 이동됨
- **404 Not Found**: 요청한 리소스를 찾을 수 없음
- **500 Internal Server Error**: 서버 내부 오류 발생

웹사이트를 이용하다 마주치는 "404 페이지"가 바로 이 응답 코드에서 비롯된 것입니다.

## 4. 정적 데이터와 동적 데이터

웹 서버의 핵심 역할은 **정적(static) 데이터**를 전달하는 것입니다.

정적 데이터란 사진, 텍스트, 동영상 파일처럼 요청에 따라 내용이 달라지지 않는 데이터를 의미합니다. 다시 말해, 클라이언트가 요청할 때마다 (파일 자체가 변경되지 않았다면) 동일한 결과값을 돌려주는 데이터입니다. 서버에 이미 저장되어 있어, 요청이 오면 그대로 전달할 수 있는 파일들이죠. HTML 파일, CSS, JavaScript, 이미지, 동영상 같은 것들이 여기에 해당합니다.

이러한 정적 데이터를 처리하는 대표적인 웹 서버 소프트웨어로는 **NGINX**, **Apache** 등이 있습니다.

그렇다면 한 가지 상황을 생각해볼까요. 만약 클라이언트가 "현재 로그인한 사용자의 계정 정보를 보여달라"고 요청하면 어떻게 될까요?

이 경우 웹 서버는 해당 요청을 직접 처리할 수 없습니다. 왜냐하면 로그인 정보 조회처럼 **사용자의 요청에 따라 결과값이 달라지는 데이터**는 웹 서버가 일반적으로 처리할 수 없기 때문입니다. 이렇게 요청 시점이나 요청 주체에 따라 내용이 달라지는 데이터를 **동적(dynamic) 데이터**라고 합니다.

동적 데이터의 대표적인 예시는 다음과 같습니다.

- 로그인한 사용자의 마이페이지 정보
- 검색어에 따라 달라지는 검색 결과
- 실시간으로 변경되는 게시판 목록
- 사용자별로 다르게 추천되는 상품 목록

이러한 동적 처리는 **WAS(Web Application Server)** 가 담당합니다. Apache Tomcat이 대표적인 WAS입니다. 실무 환경에서는 보통 다음과 같은 구조를 사용합니다.

> 클라이언트 → 웹 서버(NGINX 등) → WAS(Tomcat 등) → 데이터베이스

웹 서버는 앞단에서 요청을 받아 정적 데이터는 직접 응답하고, 동적 처리가 필요한 요청만 뒤쪽 WAS로 전달합니다. 이 구조를 알아두시면, 시스템 아키텍처 관련 질문에 한결 수월하게 대응하실 수 있습니다.

## 5. 대표적인 웹 서버 소프트웨어

앞서 언급한 NGINX와 Apache를 포함해, 실무에서 자주 사용되는 웹 서버를 조금 더 자세히 살펴보겠습니다.

**Apache HTTP Server**는 1995년에 처음 공개된 오픈소스 웹 서버입니다. 모듈 기반 구조로 확장성이 뛰어나고, 오랜 기간 사용되어 온 만큼 학습 자료와 커뮤니티가 풍부합니다.

**NGINX**는 2004년에 공개된 웹 서버로, 비동기 이벤트 기반 아키텍처를 채택했습니다. 다수의 동시 접속을 효율적으로 처리할 수 있어, 현재 많은 서비스에서 사용되고 있습니다. 리버스 프록시나 로드 밸런서 역할도 함께 수행합니다.

**Microsoft IIS(Internet Information Services)** 는 마이크로소프트가 윈도우 서버 환경을 위해 개발한 웹 서버입니다. .NET 기반 시스템과의 통합이 강점입니다.

## 6. 왜 알아야 할까요?

웹 서버 개념은 단순히 "알면 좋은" 지식이 아닙니다. 실제 업무와 직접 연결되는 기초 지식입니다.

백엔드 개발자에게는 당연히 필수 역량입니다. 프론트엔드 개발자도 배포 환경을 이해하려면 웹 서버를 피해갈 수 없고요. DevOps나 인프라 직군은 말할 것도 없습니다. 채용 공고에서 "NGINX 운영 경험", "Apache 설정 경험" 같은 문구가 자주 보이는 이유가 여기에 있습니다.

면접에서도 "클라이언트와 서버의 차이", "웹 서버와 WAS의 차이", "정적 데이터와 동적 데이터의 차이", "HTTP 동작 원리" 같은 질문이 단골로 나옵니다. 비전공자로 IT 직군에 도전하시는 분이라면, 이 개념부터 탄탄하게 잡고 시작하시는 걸 권합니다.

## 마무리

오늘은 웹 서버의 기본 개념을 정리해보았습니다. 핵심을 다시 짚어보면 이렇습니다.

- 웹 서버는 HTTP 요청을 받아 응답을 돌려주는 프로그램(또는 그 컴퓨터)이다.
- 클라이언트-서버 모델에서 "서버" 역할을 담당한다.
- 요청에 따라 내용이 달라지지 않는 **정적 데이터**를 처리하는 것이 핵심 역할이다.
- 사용자 요청에 따라 결과가 달라지는 **동적 데이터**는 WAS가 담당한다.
- 대표적인 웹 서버 소프트웨어로는 NGINX, Apache, IIS 등이 있다.

개념만 머리로 익히는 것보다, NGINX나 Apache를 직접 로컬 환경에 설치해서 간단한 페이지를 띄워보시는 걸 추천드립니다. 한 번 실습해보시면 글로만 읽었을 때보다 훨씬 명확하게 이해됩니다.

다음 글에서는 웹 서버와 WAS의 차이를 더 깊이 있게 다뤄볼 예정입니다. 오늘도 학습하시는 여러분을 응원합니다.
